receive_primitive(primitive: P_DATA) → None[source]Â¶

Process a P-DATA primitive received from the remote.

New in version 1.2.

A DIMSE message is split into one or more P-DATA primitives, which must be sent in sequential order. While waiting for all the P-DATA primitives associated with a message the encoded data is stored in message, which is decoded only when complete and converted into a DIMSE Message primitive which is added to the msg_queue.

This makes it possible to process incoming P-DATA primitives into DIMSE messages while a service class implementation is running.

primitive (pdu_primitives.P_DATA) â A P-DATA primitive received from the peer to be processed.
